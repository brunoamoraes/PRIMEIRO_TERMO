# 📋 Engenharia de Software: Levantamento de Requisitos
> **Objetivo:** Identificar, analisar e documentar as necessidades das partes interessadas (stakeholders).

---

## 1. 🎯 Requisitos Funcionais vs. Não Funcionais

A base de qualquer projeto de software reside na distinção clara do que o sistema faz e como ele opera.

*   **Requisitos Funcionais (RF):** Descrevem as funções que o sistema deve executar.
    *   *Exemplo:* "O sistema deve permitir o cadastro de novos usuários."
*   **Requisitos Não Funcionais (RNF):** Descrevem restrições ou qualidades do sistema (desempenho, segurança, usabilidade).
    *   *Exemplo:* "O sistema deve responder a qualquer consulta em menos de 2 segundos."

---

## 2. 🧠 Técnicas de Elicitação

### Brainstorming
Sessões criativas em grupo para gerar o máximo de ideias sem julgamento imediato. Útil para descobrir requisitos inovadores e necessidades ocultas.

### Entrevistas
Conversas diretas com stakeholders. Podem ser:
*   **Estruturadas:** Com roteiro fixo.
*   **Semiestruturadas:** Com roteiro, mas permitindo desvios para explorar pontos importantes.

### Prototipagem
Criação de modelos visuais (baixa ou alta fidelidade) para validar requisitos com o usuário antes do desenvolvimento real. Reduz riscos de mal-entendidos.

---

## 3. 📊 Modelagem e Diagramas
Representação visual dos requisitos para facilitar a comunicação com a equipe técnica.

*   **Diagrama de Caso de Uso (UML):** Mostra a interação dos atores (usuários) com as funcionalidades do sistema.
*   **Diagrama de Fluxo de Dados (DFD):** Ilustra como a informação transita pelos processos.
*   **BPMN:** Utilizado para modelar o processo de negócio atual e o desejado.

---

## 4. 📝 Relatórios Técnicos e Documentação
A saída do processo de levantamento é o documento formal.

### Documento de Visão
Define o escopo de alto nível e os objetivos do projeto.

### Especificação de Requisitos de Software (SRS)
O relatório técnico detalhado contendo:
1.  **Introdução:** Finalidade e escopo.
2.  **Descrição Geral:** Perspectiva do produto e funções.
3.  **Requisitos Específicos:** Detalhamento técnico de cada RF e RNF.
4.  **Critérios de Aceite:** O que define que o requisito foi atendido.

---

## 🛠️ Atividade Prática
1.  **Estudo de Caso:** Dado um cenário de uma "Biblioteca Online", divida a turma em grupos.
2.  **Brainstorming:** Cada grupo deve listar 5 funcionalidades essenciais.
3.  **Classificação:** Separar as ideias entre RF e RNF.
4.  **Esboço de Protótipo:** Desenhar em papel a tela de busca de livros (Prototipagem de Baixa Fidelidade).
